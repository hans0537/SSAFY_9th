# 05_pjt

참여자 : 박지현, 신성주, 이지은

* [박지현]

배운점 : git을 활용한 협업 작업, merge를 이용해 충동을 방지하는 법

느낀점 : 실제로 많이 쓰일 것 같은데 배워서 좋았습니다. 
        다만 처음 해보다 보니 어려운 부분들이 있었는데 성주님이 잘하셔서
        아주 좋았습니다. 배워간 것도 많습니다!
        그 뒤 구현은 어렵지 않았지만 (이미 해본 것들이라)
        병합하는 과정에서 다시 쓰고 하는 일들이 반복되어졌습니다.
        그렇지만 많은 것을 배울 수 있는 관통 프로젝트 시간이었습니다.

어려웠던 점 : git을 사용하는 작업이 익숙하지 않아서 어려웠던 부분들이 존재했고 
            병합하는 과정에서 실수로 내가 쓴 코드들을 날려버리는 것도 존재해서 어려웠습니다.

* [신성주]

배운점 : git branch를 나누어 각각의 기능을 개발은 한후 merge하는 작업
        협업 프로젝트를 git으로 효과적으로 하는 법

느낀점 : 각각의 기능을 개발하면서 다른 팀원의 기능이 조금씩 필요한 부분들이 생겼다.
        그때 마다 각각의 branch를 master에 merge를 한후에 push 하고 각각 또 pull을 하는 작업이
        생각보다 까다롭다고 생각했다. 또한 중간중간에 오류가 많이 생겨 시간이 많이 지체되었다.
        따라서 체계적으로 버전관리를 하려면 메뉴얼이나 규칙을 정한후에 깃 관리를 해야한다고 생각되었다.

어려웠던 점 : git 충돌시 해결방법. merge 할때 같은 곳에서 작업이 꼬여 수정하는 법과 브랜치가 꼬여 수정하는 법등
                생각보다 충돌이 자주 나서 다양한 오류를 맛보게 되었다.



* [이지은]

배운 점 : git branch 생성 및 활용, merge

느낀 점 : git을 통해서 개발 역할을 나누고, 어떻게 협업해야 할지 배울 수 있었다.

어려웠던 점 : 아직 git 환경에 익숙하지 않아서 어려웠던 것 같다. 충돌이 났을 때 당황하지 않고 충돌을 해결하는 방법을 배워야겠다.
